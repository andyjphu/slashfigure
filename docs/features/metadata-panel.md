# Metadata Panel

## Overview
A visible panel (side panel or bottom panel) showing the auto-generated metadata description of the current figure. Updates live as the user draws. The user can see exactly what an LLM would see.

## Behavior

### Auto-generation
- Every drawing action triggers metadata regeneration for affected elements
- The panel displays the compact text summary and/or ASCII approximation
- Updates are debounced (~300ms after last action) to avoid flicker during rapid edits

### User-modified content is protected
- If the user manually edits a line or property in the metadata panel, that line is "pinned"
- The auto-generator will NOT overwrite pinned lines
- Pinned lines are visually distinguished (different background color or icon)
- User can un-pin a line to return it to auto-generation

### Display Modes
1. **Text Summary** -- the compact natural-language description (~50 tokens)
2. **ASCII Art** -- the spatial approximation using `| . _ -` and text characters
3. **Raw JSON** -- the full scene graph (collapsible, for power users)

### Interaction
- Click an element on canvas → its metadata entry highlights in the panel
- Click a metadata entry → corresponding element highlights on canvas
- Copy button → copies metadata to clipboard (for pasting into LLM prompts)
- Export button → saves metadata as `.md` or `.json` file

## Architecture
The metadata generator is a pure function:
```
generateMetadata(sceneGraph: Element[]) → {
  textSummary: string,
  asciiArt: string,
  structuredJson: object
}
```

It runs in a requestIdleCallback or Web Worker to avoid blocking the UI. Pinned lines are stored separately and merged with generated output.

## Resolved Decisions
- **Location:** Right sidebar, under properties panel. When viewing metadata, it replaces/tabs within the properties panel context. Every element in the sidebar must justify its existence -- no filler descriptions.
- **ASCII art:** Read-only. It's a mechanical spatial render from the scene graph.
- **Token count:** Yes, exact count via `js-tiktoken` (WASM, runs in browser). Show for the selected LLM provider.
- **Display:** Tabs within the right sidebar (Text Summary / ASCII / Raw JSON). Default to Text Summary.
- **Export format:** Both Mermaid (~50 tokens) AND ASCII (~100-200 tokens) are always generated by default. Export settings include option to exclude either format. Both are kept by default for future ablation studies comparing format effectiveness.
